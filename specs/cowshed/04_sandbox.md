# Layered Sandbox

Every command cowshed executes in a workspace runs under a macOS Seatbelt (`sandbox-exec`) profile generated by
cowshed-core. The model is **start closed, widen by grant, narrow by revoke**: a fresh workspace can write only its own
mount, designated shared-cache subtrees, and temp; its egress is localhost-only. A controller — jcode's coordinator
through the Rust API, or a human/agent through the CLI — widens capabilities at runtime without restarting anything.

## Enforcement points

| Capability              | Enforced by                          | Applies                                                      |
| ----------------------- | ------------------------------------ | ------------------------------------------------------------ |
| Filesystem read/write   | Seatbelt profile                     | at the next `exec` (profiles are immutable per process tree) |
| Egress beyond localhost | cowshed-gateway per-workspace policy | immediately (gateway re-reads grants per request)            |

Both read the same grant file, so one `cowshed grant` call updates both planes.

## Baseline (closed) profile

Generated per exec (or once per supervisor launch for long-lived shells, in which case the whole process tree inherits
it — the jcode supervisor pattern). Shape:

```
(version 1)
(deny default)

;; dyld and toolchains need broad read to boot processes.
(allow file-read-data (subpath "/"))
(allow process-exec process-fork)
(allow file-map-executable)
(allow sysctl-read)
(allow pseudo-tty)
(allow process-info* (target same-sandbox))
(allow signal (target same-sandbox))
(allow mach-priv-task-port (target same-sandbox))

;; Local IPC: scoped, never blanket — a blanket unix-connect allow would let a
;; workspace reach a sibling's supervisor socket (same uid; file permissions are
;; no boundary) and run commands in its warm shells. Rule paths MUST be canonical
;; (measured: the kernel canonicalizes the connect target before matching — a
;; /tmp-spelled rule silently denies what a /private/tmp rule allows).
(allow network-outbound (remote unix-socket (path-literal "<nix daemon socket>")))
(allow network-outbound (remote unix-socket (path-literal "<own supervisor socket>")))

;; Loopback TCP — isolation rides ENTIRELY on outbound. Measured SBPL constraints
;; (see 08_testing.md): port RANGES do not parse ("invalid port in network
;; address") and hosts must be `localhost` or `*` — so the block is emitted as 16
;; literal single-port outbound allows, and the 16384-port ephemeral range cannot
;; be enumerated at all. Restricting network-bind would therefore break bind(0)
;; (measured: EPERM; the kernel does not pick a within-policy port), so bind and
;; inbound stay PERMISSIVE — dev servers, bind(0) listeners, and host-browser →
;; dev-server all just work (all measured). The implicit source-port bind on an
;; outbound connect is exempt from network-bind filtering (measured), so
;; outbound-only restriction is safe.
(allow network-bind network-inbound (local tcp "localhost:*"))
(allow network-outbound (remote tcp "localhost:<base>"))    ;; own gateway listener
(allow network-outbound (remote tcp "localhost:<base+1>"))  ;; own service ports…
;; … one rule per port through <base+15> — 16 single-port rules, never a range.
;; Everything else on loopback — 7644 (control plane), every sibling's block and
;; ephemeral listeners — falls through to (deny default) for CONNECT: a sibling
;; may bind anything, but this workspace cannot reach it (measured: EPERM).
;; Consequence of the non-enumerable ephemeral range: an in-sandbox client can
;; connect only to in-block ports and the allowed unix sockets — a tool that
;; spawns a helper server on a random port and connects back to it must be
;; pointed at block ports (PORT/COWSHED_PORT_BASE). Untested edge, tracked in
;; the kickoff experiments: which real tools this bites in practice.

;; Writable roots — the closed baseline.
(allow file-write*
  (subpath "<workspace mount>")
  (subpath "~/.cowshed/caches/sccache")
  (subpath "~/.cowshed/caches/zig")
  (subpath "~/.cowshed/caches/gradle")
  (subpath "<exec temp dir>")
  (literal "/dev/null") (literal "/dev/stdout") (literal "/dev/stderr")
  … granted write paths …)

;; Secrets: denied. SBPL is LAST-MATCH-WINS (measured, both directions, network
;; and filesystem) — these rules protect anything only because generation emits
;; them AFTER every allow. Ordering is a load-bearing invariant, not precedence.
(deny file-read* file-write*
  (subpath "~/.ssh") (subpath "~/.gnupg") (subpath "~/.aws")
  (subpath "~/.config/gh") (literal "~/.netrc")
  (literal "~/.npmrc") (literal "~/.pypirc")           ;; user-level, may hold auth
  (literal "~/.cargo/config.toml") (literal "~/.cargo/credentials.toml")
  (subpath "~/.cargo/bin")                             ;; on PATH — write = persistence escape
  (literal "~/.gradle/gradle.properties")
  (subpath "~/Library/Keychains")
  (subpath "~/.cowshed/store")                         ;; images, grant files, gateway state, logs
  … denied paths from policy …)
```

Notes:

- The broad `file-read-data` allow is required for dyld; confidentiality of specific paths is achieved by the explicit
  deny list — which works **only because profile generation emits denies last**. SBPL has no deny-beats-allow
  precedence: evaluation is last-match-wins (measured: the same rules in the opposite order leave the token readable).
  The entire secret-protection model depends on this ordering; it is asserted by a unit test (08_testing.md) that
  generated profiles place every deny after every allow. The deny list always includes cowshed's store volume
  (`~/.cowshed/store`, 01_storage.md), so no sandboxed process can touch images, grant files, or gateway credentials.
- The caches volume's **mirror** and **git** (bare-mirror) subtrees are readable but never writable from a sandbox (only
  the gateway writes layer-1 artifacts — 03_caches.md, 05_gateway.md).
- `~/.cargo` and `~/.gradle` are deliberately _not_ relocated wholesale to the cache volume — only their cache subtrees
  are (03_caches.md). The deny list above pins their config, credential, and PATH-resolved binary paths to the host
  precisely because the cache volume is sandbox-writable.
- Each workspace serves on its own block ports (`base+1 … base+15`), so dev servers that honor the port convention never
  collide — the port block _is_ the collision fix. Because bind stays permissive (macOS has no per-process network
  namespace), a sibling may still _bind_ any loopback port, including a hardcoded default; isolation holds anyway
  because **connect is the boundary**: a workspace's outbound set is its own 16 ports plus its scoped unix sockets, so
  it cannot reach a sibling's listeners — block or ephemeral — at all (measured: EPERM). Linux workspaces get a private
  loopback via netns (below), so nothing is shared and the block scheme is unnecessary there (the gateway is reached
  over a per-workspace unix socket instead).
- **Cooperative sandboxing caveat.** The block only helps tools that take port configuration. `cowshed ensure --envrc`
  exports `COWSHED_PORT_BASE=<base>` and `PORT=<base+1>` as the conventions dev servers should honor; a tool that
  hardcodes a fixed default port must be pointed at a block port (devenv port offsets derive from `COWSHED_PORT_BASE`).
  Tools that ignore the convention bind outside their block and are denied.
- `RunSandboxMode::ReadOnly` drops the workspace mount from the write set — for inspector-style commands that must
  observe without mutating.
- Read grants exist for paths outside the default-readable set matters (e.g. metadata/listing of ancestor dirs) and to
  _narrow_: a denied path can only be re-allowed by the controller removing it from the deny set, never by a grant.

## Grants

### Grant file: `<image>.grants.json`

Controller-owned, outside the workspace volume, mode 0600, on the `cowshed.store` volume (itself denied to every
sandbox). Never readable or writable from inside a sandbox.

```json
{
  "version": 1,
  "workspace": "raven",
  "revision": 7,
  "portBlock": { "base": 40976, "size": 16 },
  "updatedAt": "2026-07-11T12:34:56Z",
  "read": ["/Users/danny/Dev/shared-fixtures"],
  "write": ["/Users/danny/Dev/conloca-artifacts/raven"],
  "egress": [{ "host": "registry.npmjs.org" }, { "host": "*.github.com", "ports": [443] }, { "host": "crates.io" }]
}
```

- Paths are canonicalized on write; a grant that intersects the secret deny list is refused (exit 6 /
  `CowshedError::SandboxDenied`).
- `revision` increments on every change; execs and the gateway log the revision they enforced, making audit trails
  reconstructible.
- Egress hosts are exact names or single-level `*.` wildcards, optional port list (default 443/80).
- `portBlock` is the workspace's contiguous 16-port block from the reserved range (default 40960–49151; 05_gateway.md):
  `base` is the gateway's data-plane listener, `base+1 … base+15` are the workspace's own bindable service ports.
  Allocated at new/fork (adopt, for main), preserved across restore, never inherited by a fork. `gatewayPort` is a
  documented alias for `portBlock.base`. Controller-owned like every other field here.

### Mutation contract

Grant files are small, but they are the authority record — mutations are specified exactly:

- **Canonical set semantics.** Paths are canonicalized before comparison (symlinks resolved; the `/var` → `/private/var`
  handling generalizes); read/write/egress vectors are sorted and deduplicated; egress hosts are lowercased, wildcards
  normalized to one leading `*.` label, omitted ports normalized to the 443/80 default. Applying a delta computes a
  complete new snapshot — there is no in-place patching.
- **Locking and CAS.** Every mutation holds an exclusive `flock` on the workspace's `.lock` file (01_storage.md).
  Callers may pass an `expected_revision`; a mismatch is a conflict (exit 4 / `CowshedError::Conflict`) and the
  coordinator retries against the fresh snapshot — concurrent coordinators serialize instead of clobbering.
- **Validation before write.** A grant intersecting the secret deny list is refused (exit 6) before any bytes hit disk.
- **Crash-safe replacement.** Same-directory temporary file → write → flush → chmod 0600 → rename over the grant file →
  fsync the parent directory. A crash leaves the old file or the new file, never a torn one.
- **Revision discipline.** An effective change increments `revision` exactly once. A no-op delta (granting an
  already-present path, revoking an absent one) leaves the file byte-identical, the revision unchanged, and reports the
  current set; revoking a nonexistent item is a no-op, not an error.

### Widening and narrowing

```
cowshed grant  <ws> [--read <path>]… [--write <path>]… [--egress <host[:port]>]…
cowshed revoke <ws> [--read <path>]… [--write <path>]… [--egress <host>]… [--all]
```

Rust API: `Workspace::grant(delta)` / `Workspace::revoke(delta)` — this is the generalization of jcode's
swarm-coordinator `grant_write_paths`: the coordinator starts workers closed and opens paths and egress as the task
proves the need.

Propagation:

- **Egress**: immediate. The gateway resolves the workspace by its token and consults the current grant file per
  request.
- **Filesystem**: at the next exec. Every `ExecRequest` snapshots the grant file and the profile is regenerated with the
  snapshot; long-lived supervisor trees launched before a grant continue under their launch profile until relaunched
  (cowshed surfaces the enforced revision so controllers know). Revocation likewise binds at the next exec — controllers
  that need hard cuts kill the affected process tree (`cowshed exec` reports PIDs for this purpose).

## Exec pipeline

`cowshed exec <ws> [--ro] -- cmd…` / `Workspace::exec(ExecRequest)`:

1. Resolve workspace (marker), verify mounted (heal via ensure logic if not).
2. Snapshot grants; validate cwd against policy; refuse commands whose cwd escapes the mount in rw mode.
3. Compose env: identity + cache wiring (03_caches.md) + the workspace's own `.envrc` via fail-closed `direnv export`
   (see "devenv / Nix inside the sandbox") + caller env filtered through an allowlist (build-configuration variables
   only — never `*_TOKEN`, `*_SECRET`, `AWS_*`, etc.).
4. Generate the profile, write it to a private temp file, spawn `sandbox-exec -f <profile> /usr/bin/env <env…> <cmd…>`
   with cwd inside the mount.
5. Stream stdout/stderr through unmodified; report exit code, PIDs, and enforced grant revision.

## Sandbox-denial evidence

Exit 6 / `SandboxDenied` is a promise that cowshed _diagnosed_ a denial, so it may be produced only from authoritative
evidence — never inferred from output text:

1. **Pre-spawn validation** — a cwd, path, or mode that policy refuses before any process runs.
2. **Profile application failure** — `sandbox-exec` / Landlock setup errors.
3. **Gateway policy denials** — always authoritative: the 403 body carries the exact resolving grant (05_gateway.md) and
   the CLI surfaces it verbatim.
4. **errno discrimination (measured)** — a Seatbelt denial surfaces as **EPERM (errno 1)** on both `open` and `connect`,
   cleanly distinguishable from an allowed-but-unserved connection (**ECONNREFUSED, errno 61**). This in-band signal is
   authoritative enough for exit 6 to stand on its own when the failing syscall and target are known (e.g. via the
   gateway or pre-flight probes cowshed itself performs).
5. **Unified-log correlation (macOS) — enhancement, unverified.** Seatbelt violations are believed to reach the unified
   log with pid, operation, and target, which would let cowshed-shell match a job's pid tree + time window (11_shell.md)
   to produce `denial.kind` and `suggested_grant` for arbitrary child syscalls. This could not be verified from a
   sandboxed session (the log store itself is permission-gated) and remains an open item — verify from an unsandboxed
   controller context (kickoff experiments). The design does not depend on it: sources 1–4 stand alone. The Linux
   analogue is Landlock audit/LSM notifications where the kernel provides them.

The SBPL mechanics this profile depends on are measured, not assumed (08_testing.md): port ranges do not parse;
last-match-wins ordering; implicit bind-on-connect exempt from `network-bind`; `bind(0)` works under permissive bind and
fails EPERM under restricted bind.

When none of these produce evidence, the child's ordinary exit status passes through unchanged — a failure is not a
denial just because its output mentions one. Grants are **never** synthesized from stdout/stderr text: merged-output
heuristics (jcode's current `operation not permitted` / `deny(` string scanning) are spoofable by any child process,
produce both false positives and false negatives, and cannot recover the denied path. They are explicitly rejected as an
evidence source.

## devenv / Nix inside the sandbox

Workspaces are devenv/direnv projects: the cloned repo carries `.envrc`, `devenv.nix`, `devenv.lock`, and its gitignored
`.devenv` state — all in-image, all workspace-keyed. The closed baseline makes devenv work by construction, plus four
guarantees cowshed owns:

- **Store reads**: `/nix/store` is covered by the broad `file-read-data` allow; nothing to grant.
- **Builds and substitution go through the nix daemon**, reached over its unix socket (a scoped canonical-path
  `unix-socket` allow in the baseline) and running outside the sandbox. Multi-user (daemon) Nix is a requirement;
  single-user installs, where the client writes `/nix/store` directly, are unsupported. Substitution from binary caches
  is therefore an off-gateway egress path — accepted as a trusted-mediator channel (root daemon, signature-checked
  caches), documented, never proxied. GC-root registration for `.devenv` profiles is likewise daemon-side; no grant
  needed.
- **Nix client state**: the closed baseline's writable roots additionally include `~/.cache/nix` and
  `~/.local/state/nix` (eval/fetcher caches, profiles state). These are small SQLite files; they stay on the host and
  are shared across workspaces like any concurrency-safe cache.
- **direnv trust is path-keyed**, so every clone's `.envrc` is untrusted at birth. cowshed (unsandboxed) writes the
  direnv allow entry for `<mount>/.envrc` at new/fork/restore — and `cowshed ensure` re-asserts it when healing.
  `cowshed exec` then loads the environment exactly as an interactive shell would: when a nearest `.envrc` exists, the
  command is wrapped fail-closed in `direnv export` (missing direnv or a failed export is an error, never a silent skip)
  — jcode's `command_with_nearest_envrc`, minus its secret-var filtering: inside a cowshed workspace the full exported
  environment is safe by the no-secrets invariant.

  Two mechanisms here, not one — _trust_ and _evaluation cost_ are independent. Trust (authorization to evaluate at all)
  is path-keyed, and cowshed re-asserts it per clone as above. That evaluating a fresh clone's `.envrc` is _cheap_ is a
  separate claim resting on the in-image `.devenv`/Nix eval state arriving via CoW; it is a listed verification item
  (08_testing.md, kickoff experiments), not an assertion this spec depends on.

Port collisions between workspaces are handled by the per-workspace port block, not left to the user: `devenv up` and
dev servers bind ports derived from `COWSHED_PORT_BASE` (the block base), so two workspaces running the same service set
land on different ports and both are reachable from the host browser. The residuals are the cooperative-sandboxing
caveat above (a tool that hardcodes a port can still bind it — bind is permissive — but collides with a sibling doing
the same and is unreachable from its own workspace's clients), and the in-block-only rule for in-sandbox clients: a
workspace process can connect only to its own block ports, so service-to-service traffic inside a workspace must also
ride block ports. On Linux the netns makes both moot: each workspace's loopback is private.

## Linux enforcement (ZFS substrate)

On Linux the model is identical — start closed, widen by grant, narrow by revoke, one grant file feeding both planes —
only the enforcers change. Nothing above the enforcement layer differs: same `<image>.grants.json` schema
(09_substrates.md: `~/.cowshed/store/…` on Linux too), same revision semantics, same propagation (filesystem at next
exec, egress immediate via the gateway), same exit code 6 on a grant that intersects the deny set.

| Capability              | macOS                           | Linux                              |
| ----------------------- | ------------------------------- | ---------------------------------- |
| Filesystem read/write   | Seatbelt `sandbox-exec` profile | Landlock ruleset                   |
| Egress beyond localhost | gateway per-workspace policy    | gateway policy + network namespace |

- **Filesystem: Landlock.** cowshed-core compiles the same closed baseline + grant snapshot into a Landlock ruleset
  (unprivileged, no helper needed) applied to the exec'd process before it runs the command: read on the broad system
  roots and toolchains, read/write on the workspace mount, designated cache subtrees, and temp, plus granted paths.
  Secret paths are enforced by _omission_ (Landlock is allowlist-shaped — a path with no rule is inaccessible), which is
  structurally stronger than Seatbelt's ordering-dependent deny-last emission; the deny list still drives which
  ancestors cowshed refuses to add even on grant. `RunSandboxMode::ReadOnly` drops the mount's write access. Landlock
  ABI is probed at startup; cowshed requires ABI ≥ 3 (fs + refer) and refuses to run unsandboxed on kernels below it
  unless `COWSHED_NO_SANDBOX=1` is set explicitly (same escape hatch, same audit note as macOS).
- **Egress: per-workspace unix socket (no netns needed).** The gateway's per-workspace data-plane listener is a unix
  socket bind-mounted / path-scoped into the workspace — the transport Anthropic's
  [sandbox-runtime](https://github.com/anthropic-experimental/sandbox-runtime) uses, whose Linux design removes the
  network namespace entirely and serves its proxies over unix sockets (no veth, no DNAT). The workspace has no TCP
  egress at all; all network exits through that socket, and the gateway applies the per-workspace allowlist exactly as
  on macOS. Because each workspace has a private loopback (a fresh netns is still used to isolate loopback so dev
  servers don't collide, but it needs no routing plumbing), the port-block scheme is a macOS mechanism only — sibling
  services neither collide nor are reachable here.
- **direnv/devenv, secrets scanner, marker/token handling**: unchanged — they are substrate- and OS-independent pure
  logic plus the daemon-socket Nix access described above.

## cowshed-escape-tests

A regression harness (modeled on jcode's `jcode-sandbox-escape-bash-*` crates) that runs adversarial commands through
the real exec pipeline and asserts denial. The macOS suite runs under Seatbelt; the same cases run on Linux under
Landlock, plus Linux-specific escapes:

- write outside granted roots (absolute, relative, `..`-traversal, symlink pivot, hardlink);
- read `~/.ssh`, `~/.aws`, Keychains, the `cowshed.store` volume, another workspace's mount;
- tamper with own grant file or marker via crafted paths;
- connect to a sibling workspace's supervisor socket (must be refused by the scoped unix-socket allow — including via a
  non-canonical `/tmp`-alias path);
- connect to a sibling's block ports and to a sibling's ephemeral-bound listener (both EPERM — isolation is
  outbound-enforced; sibling binds are not prevented);
- assert profile ordering end-to-end: a secret-deny path stays denied even when a grant allows an ancestor (denies
  emitted last — the last-match-wins invariant);
- write to `~/.cargo/bin`, cargo config/credentials, or `~/.gradle/gradle.properties` — directly and via the
  sandbox-writable cache volume;
- egress to a non-granted host directly and via helper processes (`curl`, `nc`, `python`);
- write through `/tmp` symlinks into denied paths; abuse `file-map-executable` scratch dirs;
- verify revocation binds at next exec and ReadOnly mode rejects mount writes.

Linux-specific cases: bind-mount a denied path into a granted root, `/proc/<pid>/root` and `/proc/<pid>/cwd`
reach-arounds, `unshare`/`setns` attempts to leave the netns, abstract-namespace and filesystem unix sockets reaching a
non-gateway listener, and `connect(2)` to a non-gateway TCP port. Each escape found in production becomes a permanent
case here. The suite is a release gate on both OSes.

## Tradeoffs

**Per-command wrapping vs launch-time supervisor sandboxing.** cowshed supports both, like jcode: launch-time wrapping
(one profile for a shell supervisor's whole tree) is the cheap common path; per-exec wrapping is what makes grant
snapshots enforceable and ReadOnly narrowing possible. The cost of launch-time trees — grants bind only at relaunch — is
surfaced, not hidden.

**VM isolation rejected (v1).** Seatbelt confines file writes and egress but shares the kernel; a kernel exploit escapes
it. The alternatives fail the fleet: Apple containers run only Linux guests, and macOS guests are capped at two
concurrent VMs by Virtualization.framework. cowshed's threat model is semi-trusted agents running the user's own code
under supervision — confinement, not hostile-code isolation — and the spec says so plainly.

**String-scanning commands for sensitive paths rejected.** Blocking commands that _mention_ `~/.ssh` is bypassable by
construction (encoding, indirection). cowshed relies on kernel-enforced deny rules; command text is never
security-relevant.

**Prior art.** The Seatbelt-plus-host-proxy shape is the same one Anthropic ships in
[sandbox-runtime](https://github.com/anthropic-experimental/sandbox-runtime) for Claude Code: OS-level enforcement, no
container, egress mediated by host-side proxies. Its transport choice — proxies listening on unix sockets passed into
the sandbox — is the model cowshed adopts for the Linux netns leg above.
