Every op receives a span context (`ctx`). It is the API for the currently executing span.

## `ctx.tag`

Set attributes on this span. Each setter returns the tag object, so calls chain. `with({...})` sets several at once.

```ts
ctx.tag.userId('ada').itemCount(3);
ctx.tag.with({ route: '/cart', status: 200 }).itemCount(5);
```

Field names and value types come from your `logSchema`.

## `ctx.log`

Append a timestamped event. Levels: `info`, `debug`, `warn`, `error`. The message is a template; attach values via the
same fluent chain.

```ts
ctx.log.info('processing {{itemCount}} items').itemCount(3);
ctx.log.warn('slow path');
```

Beyond your schema fields, a log entry also exposes the reserved system-column setters — including `.line(n)` (source
line) and `.uint64_value(value)`, which writes the 64-bit `uint64_value` column (its argument is a `bigint`):

```ts
ctx.log.info('scanned rows').uint64_value(1_234_567_890_123n);
```

## `ctx.span(name, opOrFn)`

Open a child span. Pass an op, or an inline function that receives the child context. Returns a `Promise<Result>`.

```ts
await ctx.span('validate', validateOp);
await ctx.span('save', async (child) => child.ok({ id: 1 }));
```

## `ctx.spanSync(name, op)`

Open a **synchronous** child span that runs a **no-argument op** (`Op<…, []>`) with a synchronous body. Unlike
`ctx.span`, `spanSync` returns the op's `Result` directly — no `Promise`, no `await` — and does no async retry or
backoff. Use it when the child work is synchronous and can't suspend.

```ts
// A no-arg op with a synchronous body.
const parseConfig = defineOp('parse-config', (child) => {
  child.tag.step('parse');
  return child.ok(JSON.parse(raw));
});

const parsed = ctx.spanSync('parse', parseConfig);
```

## `ctx.setScope(values)`

Set default values for this span that are **inherited by child spans**. Call once for correlating context.

```ts
ctx.setScope({ requestId: 'req-1', userId: 'ada' });
```

## `ctx.ff`

Access feature flags (present only when the op context declares `flags`). Sync flags are properties; async flags use
`get()`. See [Feature flags](/guides/feature-flags/).

```ts
if (ctx.ff.advancedValidation) {
  /* ... */
}
const experiment = await ctx.ff.get('experiment');
```

## `ctx.deps`

Access dependency op groups declared in `deps`. Invoke a dependency op as a child span.

```ts
await ctx.span('auth-check', ctx.deps.auth.checkAuth);
```

## `ctx.ok(value)` / `ctx.err(error)`

Complete the span. `ctx.ok` returns success; `ctx.err` returns a failure with a typed error. Both write to the
span-end row (row 1) and carry the same fluent setters as `ctx.tag` — individual field setters, plus `with({...})`
for several at once — along with `.message('...')` and `.line(n)`.

```ts
return ctx.ok({ id: 'user-1' }).status(200);
return ctx.err(NOT_FOUND({ id })).with({ userId: id, status: 404 });
```

Reach for the result's setters — not `ctx.tag` — for anything that isn't known until the outcome is (an HTTP status
that differs by branch, for example): tagging it up front would record that value even on a path that never reaches
it. See [Integrate with an HTTP framework](/guides/integrate-with-http-framework/).

## `ctx.buffer`

The span's underlying buffer. Useful in tests for querying the span tree (`querySpan(ctx.buffer)`) and for reading
`ctx.buffer.trace_id`.

## User context properties

Any properties declared in `defineOpContext({ ctx: { ... } })` are available directly on `ctx` (for example
`ctx.requestId`, `ctx.env`). They are inherited by child spans.
