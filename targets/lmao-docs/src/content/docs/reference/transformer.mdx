`@smoothbricks/lmao-ttsc` is a **separate companion package** (not a subpath of `@smoothbricks/lmao`) that provides
LMAO's compile-time optimizer. It is a native **ttsc / tsgo compiler plugin** (a Go plugin loaded through
[`ttsc`](https://www.npmjs.com/package/ttsc)) that rewrites your traced code at build time to remove runtime dispatch
and setup wherever static proof is available.

:::caution[Entirely optional]
No documented `@smoothbricks/lmao` API requires this plugin. Your code compiles and runs identically without it — you
simply forgo automatic source-line numbers and the hot-path lowering below. It is a pre-1.0, advanced build integration;
treat the emitted shape as an implementation detail, not a contract.
:::

```bash
bun add -d @smoothbricks/lmao-ttsc
```

## Integration

The plugin is a **native ttsc descriptor**, not a TypeScript compiler-API transformer you register by hand. `ttsc`
discovers it automatically from the package's `ttsc.plugin` field once it is a direct dependency — you do not wire up a
second transformer implementation.

For **`Bun.build`**, use the package's adapter export:

```ts
import ttsc from '@smoothbricks/lmao-ttsc/bun';

await Bun.build({
  entrypoints: ['src/index.ts'],
  outdir: 'dist',
  plugins: [ttsc()],
});
```

For Bun's **runtime loader** (`bun run` and `bun test`), register the same adapter once in `bunfig.toml`:

```toml
[test]
preload = ["@smoothbricks/lmao-ttsc/bun-register"]
```

Both entrypoints read the nearest TypeScript project and apply its configured and direct-dependency ttsc plugins. The
`./bun-register` export is intentionally a package side effect; the `./bun` build adapter itself is side-effect free.

## What it rewrites

At compile time, for code written against the shipped `ctx` surface:

- **Source-line injection** — `.line(N)` is added to `ctx.log.*`, `ctx.ok(...)`, and `ctx.err(...)` with the call-site
  line number (inserted before any further chained setters; chains that already carry `.line(...)` are left alone).
- **Monomorphic `span()` lowering** — `ctx.span('name', op, ...args)` is lowered to a fixed-arity, Promise-preserving
  internal call so the hot path avoids polymorphic dispatch.
- **Tag inlining** — a `ctx.tag.*` chain (and `ctx.tag.with({...})`) used as a statement is unrolled to direct columnar
  buffer writes, so tagging costs no method calls on the hot path.
- **Destructured-context (`__ctx`) rewrite** — hoists context members at call sites for tighter emitted code.
- **Metadata injection** and **packed runtime hints** — build metadata and small runtime flags are baked in, and log
  templates are stored as private `u16` values.

## Caveats

- The plugin's contract is **the transformed output its test suite asserts**, not any particular emitted source — don't
  depend on the exact shape.
- Automatic `ctx.span(...)` → synchronous `spanAutoN(...)` lowering is **intentionally disabled**: a synchronous return
  would change the public Promise API's observable microtask scheduling.
- The "global vocabulary ABI" described in the design is a future clean-cutover target, **not** shipped behavior; shipped
  Op-local log-template IDs remain lexical `u16` values.

See the design spec `specs/lmao/01o_typescript_transformer.md` for the full transformation list and rationale.
