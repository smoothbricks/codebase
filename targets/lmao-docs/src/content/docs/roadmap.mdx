The rest of these docs cover the shipped `@smoothbricks/lmao` core. This page collects two things: **companion packages**
that ship separately and extend the core, and the directions that are **still on the roadmap** — listed for context, not
yet for use.

## Companion packages (shipped separately)

These are real, installable packages published alongside the core. They are pre-1.0, so treat their surfaces as still
settling, but they exist today.

### Query engine & inspector

Trace analysis — an in-browser SQL query engine over exported Arrow data, plus historical (HTTP) and live
(WebSocket/SSE) data sources — ships as **`@smoothbricks/lmao-inspector`**. See the
[Inspector reference](/reference/inspector/). On the writer side you produce the Arrow it consumes with the shipped
[Convert a trace to Arrow](/guides/convert-to-arrow/) and [Persist traces to SQLite](/guides/persist-to-sqlite/) APIs.

### TypeScript transformer

The build-time optimizer (source-line injection, tag/log/result inlining, monomorphic span lowering, and the
destructured-context `__ctx` rewrite) ships as **`@smoothbricks/lmao-ttsc`** — a native ttsc/tsgo compiler plugin. It is
entirely optional — no documented API requires it — and is an opt-in performance/DX optimization. See the
[Transformer reference](/reference/transformer/).

### Cloudflare trace sink

Sink adapters for streaming trace rows on Cloudflare (a diagnostic lane over Pipelines/Queues and a billing-grade lane)
ship at the **`@smoothbricks/lmao/cloudflare`** subpath. This one is **partially implemented** — the sink tracers and
transports ship, but the collector service they target is still a design. See
[Cloudflare Workers + D1](/guides/cloudflare-workers-d1/#diagnostic-trace-sink-experimental).

## Still on the roadmap

:::caution
The items below are not available as a shipped API. Don't write code against them.
:::

### AI-agent / MCP integration

Exposing traces to AI agents over a protocol such as MCP is an idea under consideration, not a shipped capability.

### Inspection CLI (`lmao-inspect`)

A small, local-first terminal tool for reading Arrow trace segments on disk — `scan` / `tail -f` / `query` / `trace`
over the tracer-agnostic `lmao-query` selector surface. It is **unbuilt** (design only) and is distinct from the
browser-based Inspector above; consumers would wrap it with domain-specific verbs.

### Rust core

A Rust reimplementation of the columnar core (`packages/lmao-rs`: buffers, Arrow conversion, the query engine, and the
WASM allocator) is in progress. It backs the AOT/WASM path but is not a separate public API you install today — the
shipped surface remains the TypeScript `@smoothbricks/lmao`.

### Partially built

A few capabilities are underway but incomplete — present in the design and partly in the code, not yet a stable API:

- **Op-metrics collection** — the reserved metric entry types and the `uint64_value` writer exist, but automatic
  per-op accumulation is not wired up.
- **Runtime column promotion** — lazy→eager column promotion is only half-built.
- **The Cloudflare collector service** — the billing-grade collector the sink adapters target is not deployed; a
  `FakeCollectorClient` ships so the lane composition is testable.

---

If you need one of the roadmap capabilities today, build on the shipped primitives: Arrow conversion, SQLite/D1 sinks,
the companion inspector package, and the trace-testing query API.
